1. Model Eloquent / Skema Database (Infrastruktur)
Kamu perlu memastikan Model Eloquent Laravel kamu sudah mencerminkan 7 tabel baru tersebut beserta relasi-relasinya.

Membuat/menyesuaikan model: ProductModel, ProductImageModel, ProductVariantModel, ProductAttributeModel, dll.

Mengatur relasi di dalam model (misal: ProductModel memiliki relasi hasMany ke ProductImageModel).

2. Data Mapper (Infrastruktur)
Buat sebuah kelas khusus (misal: ProductMapper.php) yang bertanggung jawab untuk mengubah (hydration):

Dari Database ke Domain: Mengubah hasil query Eloquent/DB menjadi instance objek Entity Product (termasuk memasukkan array ProductImage dan ProductVariant ke dalamnya).

Dari Domain ke Database: Mengubah objek Entity Product menjadi array mentah siap simpan saat ada fungsi Create atau Update.

3. Repository Implementation (Infrastruktur)
Mengimplementasikan interface repository yang ada di domain (misal: ProductRepositoryInterface). Di dalam kelas EloquentProductRepository.php ini, kamu akan menulis query DB::table() atau ProductModel::with(...) untuk mengambil data produk secara utuh (Aggregate), lalu melewatkannya ke Data Mapper sebelum dikembalikan ke Use Case.

4. Application Layer (Use Cases & DTOs)
Setelah infrastruktur siap, kamu tinggal menyesuaikan Use Case yang menggunakan data produk ini, contohnya:

CreateProductUseCase (memastikan input gambar baru dan status enum tervalidasi).

GetProductDetailUseCase (memastikan data gambar ikut ter-render di API).

Rekomendasi Langkah Selanjutnya:
Apakah kamu ingin kita buatkan Model Eloquent untuk tabel-tabel baru ini, atau langsung lompat ke pembuatan ProductMapper untuk merakit objek Product beserta ProductImage yang baru?
