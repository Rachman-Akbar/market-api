# Catalog/README.md
#   Penjelasan umum tentang domain Catalog, tujuan, dan guideline penggunaan folder.

# Application/
#   Berisi logika aplikasi (use case, DTO, service) yang mengatur proses bisnis utama.
#   - DTOs/: Data Transfer Object, struktur data untuk input/output antar lapisan aplikasi.
#   - UseCases/: Implementasi proses bisnis utama (misal: CreateProduct, GetCategories).

# Domain/
#   Berisi inti domain bisnis, entitas, dan kontrak repository.
#   - Entities/: Kelas yang merepresentasikan objek utama domain (Product, Category, Store, dsb).
#   - Repositories/: Interface/kontrak repository yang mendefinisikan operasi data domain.

# Infrastructure/
#   Berisi implementasi teknis, persistence, integrasi eksternal, dan service.
#   - Persistence/: Implementasi penyimpanan data (database, mapper, model, repository).
#     - Mappers/: Konversi antara model database dan entitas domain.
#     - Models/: Representasi tabel database (Eloquent Model).
#     - Repositories/: Implementasi repository berbasis database (Eloquent).
#   - Providers/: Service provider untuk dependency injection, binding, dsb.
#   - Services/: Service teknis (misal: integrasi API eksternal, helper).

# Presentation/
#   Berisi layer presentasi (API, HTTP, View).
#   - routes.php: Definisi routing endpoint untuk domain Catalog.
#   - Http/: Layer HTTP (Controller, Request, Resource).
#     - Controllers/: Meng-handle request, memanggil use case, dan mengembalikan response.
#     - Requests/: Validasi dan otorisasi input dari user.
#     - Resources/: Transformasi data domain ke format response API (JSON, dsb).
